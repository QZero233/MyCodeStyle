# Git Commit 风格规范

## 1. Commit Message 格式

提交信息的第一行（subject）必须遵循以下格式：

```
<type>(<scope>): <description>
```

其中 `description` 部分必须使用**中文**来写。

### type 常用取值

- `feat`: 新功能
- `fix`: 修复 bug
- `refactor`: 重构（既非新功能也非修 bug）
- `docs`: 文档变更
- `style`: 代码格式调整（不影响逻辑）
- `test`: 测试相关
- `chore`: 构建/工具/依赖等杂项

### scope

用简短的英文标识本次变更影响的模块或领域。

---

## 2. Commit Body 要求

提交信息的主体部分必须写清楚以下两点：

### 2.1 本次修改的主要内容，要解决的问题是什么

说明为什么要做这次提交，背景是什么，解决了什么问题或实现了什么需求。

### 2.2 每个文件都做了哪些修改，为什么做这些修改

逐文件列出变更内容，并解释每个文件被修改的原因——不是重复 diff 内容，而是说明意图和决策理由。

---

## 3. 示例

```
feat(auth): 添加基于 JWT 的登录认证功能

本次修改为系统引入了用户登录认证机制。之前系统缺少身份验证，
所有接口均为匿名访问，存在安全隐患。通过引入 JWT 令牌机制，
实现了无状态的用户认证。

- src/auth/jwt.rs: 新增 JWT 编码与解码逻辑，使用 HS256 算法签名
- src/auth/middleware.rs: 新增认证中间件，拦截请求并校验令牌有效性
- src/handlers/login.rs: 新增登录接口，验证用户名密码后签发令牌
- Cargo.toml: 添加 jsonwebtoken crate 依赖
```
